# 一秒翻盘 / flip-second — MVP Review Checklist

适用场景：同日交付、移动优先、微信内打开的 H5 小游戏原型。
目标：先保证“能打开、能看懂、能玩完、愿意再来一局”，再谈 polish。

---

## 1) Must-pass checklist

这些项不过，就不建议发给真实用户测试。

### A. 打开与进入

- [ ] 微信内打开后，首页在 3 秒内出现核心内容，不出现长时间白屏
- [ ] 首屏不依赖用户滚动才能看到主按钮（开始/继续/再来一局）
- [ ] 首屏标题能让人 3 秒内理解这是什么游戏
- [ ] 首屏有一句超短规则说明，第一次打开不用猜玩法
- [ ] 页面没有明显报错、资源 404、布局错位、文字重叠
- [ ] 弱网下至少能先显示骨架/文案，不至于整页空白

### B. 核心玩法可理解

- [ ] 用户第一次进入，不看别人演示，也知道第一步该点哪里
- [ ] 核心目标清楚：玩家要追求什么、何时算赢/输/翻盘
- [ ] “一秒翻盘”的主题感在前 10 秒内能被感知到，不只是名字好听
- [ ] 关键状态变化有即时反馈：点击后有视觉反馈、分数/倒计时/局势变化可见
- [ ] 失败或成功原因说得明白，不让人一脸懵

### C. 单局流程闭环

- [ ] 从进入到开始游戏，最多 1 次主要点击即可开局
- [ ] 单局时长可控，建议 10–30 秒内能完成一次完整体验
- [ ] 游戏结束后，结果页能明确告诉用户：结果、得分/表现、下一步
- [ ] 结束后有明显的“再来一局”按钮，不需要回退或重进页面
- [ ] 再来一局流程顺畅，不残留上一局状态

### D. 移动端可用性

- [ ] 主要点击区域足够大，手指易点，避免误触
- [ ] 关键按钮之间有间距，不挤在一起
- [ ] 文字在常见手机宽度下可读，不需要放大看
- [ ] 页面适配窄屏，至少覆盖 375px 左右宽度，不横向溢出
- [ ] 长屏手机上不显得内容过散，短屏手机上不出现关键按钮被遮挡
- [ ] 不依赖 hover、右键、键盘等桌面交互

### E. 微信/H5 兼容基础线

- [ ] 微信内置浏览器中可正常运行
- [ ] iPhone Safari / 微信 WebView 下无明显卡死、按钮失灵、音频异常
- [ ] Android 微信 WebView 下无明显字体抖动、定位漂移、点击延迟
- [ ] 页面刷新后不会进入坏状态
- [ ] 从聊天会话多次打开链接，状态不会随机错乱

### F. 反馈与情绪价值

- [ ] 每次关键操作都有反馈：按下、命中、失败、翻盘、结算
- [ ] 成功/失败情绪明显，至少有一种让人“哦，有感觉”的反馈
- [ ] 结果页文案不是纯技术提示，能激发“再试一次”
- [ ] 没有过度打扰的弹窗、授权、复杂表单

### G. 基础稳定性

- [ ] 连续玩 5 局不出现明显性能衰减
- [ ] 连点/狂点时不会出现重复结算、界面锁死、分数异常
- [ ] 资源加载失败时有兜底，不至于整局不可玩
- [ ] 倒计时、分数、局势值等核心数据不会出现 NaN、负数乱跳、显示错位
- [ ] 埋点/日志失败不影响游戏主流程

## 2) Stretch polish checklist

这些项不是当天 MVP 必需，但做到了会明显更像“可发给人”的产品。

### A. 首屏更抓人

- [ ] 标题、副标题、主 CTA 形成一个清晰视觉层级
- [ ] 开场 1 个小动效/局势动画，帮助理解“翻盘”主题
- [ ] 首屏有一句带情绪的 hook，而不只是功能说明

### B. 玩法说明更聪明

- [ ] 规则说明控制在 1–2 句话，或拆成 3 个超短提示
- [ ] 首局带轻引导，第二局开始自动收起，不打扰熟悉用户
- [ ] 失败后给出简短提示：你差在哪、下一次怎么翻

### C. 手感更爽

- [ ] 按钮按下态、轻震感（若环境支持）、音效反馈统一
- [ ] 关键瞬间有节奏变化，比如倒计时压迫感、翻盘瞬间增强
- [ ] 动效时长短而利落，不拖沓

### D. 结果页更会留人

- [ ] 结果页有一句会让人想截图/转发的文案
- [ ] 根据表现给不同结果文案，而不是永远同一句
- [ ] 提供“再来一局”和“分享给朋友挑战”两个清晰动作

### E. 细节一致性

- [ ] 颜色语义统一：危险、机会、成功、失败一眼能懂
- [ ] 图标、按钮圆角、阴影、字号风格一致
- [ ] 全局文案统一口吻，不一会儿像游戏，一会儿像后台系统

## 3) Likely failure risks

这些是同日 MVP 最容易翻车的点，优先盯。

### 高风险

1. **首屏看不懂**
   - 表现：用户打开后不知道该干嘛，几秒内退出
   - 常见原因：标题抽象、规则太长、主按钮不突出

2. **“翻盘感”没有做出来**
   - 表现：玩是能玩，但没有逆转瞬间，主题落空
   - 常见原因：节奏太平、胜负变化不明显、缺少关键反馈

3. **移动端按钮误触/难点**
   - 表现：想点 A 点到 B，或按钮太小、太靠边
   - 常见原因：按桌面尺寸做、缺少真机测试

4. **结束页不给下一步**
   - 表现：用户结束后愣住，不知道重开或分享
   - 常见原因：结果页只展示结果，没有明确 CTA

5. **微信 WebView 表现不稳定**
   - 表现：音频不响、fixed 定位异常、动画卡顿、字体渲染怪
   - 常见原因：只在桌面浏览器调试，没有微信内实测

### 中风险

6. **节奏拖**
   - 表现：开局前说明太久、过场太长、单局太慢
   - 后果：用户到高潮前就流失

7. **状态重置不彻底**
   - 表现：第二局继承上一局分数/倒计时/动画状态
   - 后果：用户不信任结果

8. **弱网资源阻塞**
   - 表现：图大、音频大、首屏资源过多导致白屏
   - 后果：分享出去第一印象直接没了

9. **文案像系统，不像游戏**
   - 表现：提示语机械、生硬、缺乏情绪
   - 后果：可玩但不想玩第二次

10. **过度追求动效导致卡顿**
   - 表现：掉帧、发热、输入延迟
   - 后果：爽感没做出来，反而显得廉价

## 4) Quick test script for human mobile check

给实际拿手机的人，按这个脚本 5–10 分钟走一遍就够。

### 测试前准备

- 用 1 台 iPhone 和 1 台 Android（如果只能测一台，优先你目标用户常用机型）
- 尽量在微信里直接打开链接，不只用系统浏览器
- 开启正常网络；若有时间，再补 1 次弱网环境测试

### 测试脚本

#### Round 1：第一次打开（30 秒判断生死）

1. 在微信里点开链接
2. 不看说明，直接观察首屏 5 秒
3. 问自己：
   - 我知道这是什么吗？
   - 我知道该点哪里吗？
   - 我想不想试一下？
4. 点击开始，进入第一局

**通过标准：** 5 秒内知道“这是什么 + 点哪里开始”。

#### Round 2：首局体验（看懂没有）

1. 完整玩第一局
2. 不要反复研究界面，按普通用户直觉玩
3. 结束后记录：
   - 哪一刻我明白规则？
   - 哪一刻我觉得有趣/没劲？
   - 我知道自己为什么赢/输吗？

**通过标准：** 第一局结束后，能用一句话说出玩法。

#### Round 3：再来一局（看留存）

1. 点击“再来一局”
2. 连续玩 3 局
3. 观察：
   - 有没有状态残留
   - 有没有卡顿加重
   - 我会不会自然再点一次

**通过标准：** 连续 3 局流程顺，不烦，不出错。

#### Round 4：极端手法（看抗压）

1. 快速连点主按钮
2. 在倒计时快结束时疯狂点
3. 结束瞬间连续点“再来一局”
4. 若有弹窗/分享层，快速开关几次

**通过标准：** 不重复结算、不死机、不黑屏、不出现怪状态。

#### Round 5：可读性与手感（看移动端）

1. 单手操作一局
2. 左右手各试一次
3. 在自然持机姿势下看：
   - 按钮好不好点
   - 字看不看得清
   - 关键内容会不会被手挡住
   - 有没有想点却不敢点的小按钮

**通过标准：** 主流程可以单手顺畅完成。

#### Round 6：分享感（可选）

1. 玩出一个结果
2. 看结果页是否让你想发给朋友
3. 如果不想，问：差的是成就感、文案，还是视觉记忆点？

**通过标准：** 至少有一个瞬间让人觉得“这个发出去有点意思”。

## 建议的当天验收结论模板

评审时可以直接用这 3 句做口头结论：

- **能不能发：** 能 / 不能
- **最大问题：** 首屏理解 / 手感 / 节奏 / 稳定性 / 微信兼容
- **当天最值的一刀：** 优先改一个最影响分享测试的问题

如果只能改 3 件事，优先顺序建议是：
1. 首屏是否一眼看懂
2. 单局是否 10–30 秒内形成“翻盘感”
3. 结束后是否强力引导“再来一局”
